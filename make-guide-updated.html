What is Make.com?

Make (formerly Integromat) is a powerful visual automation platform that lets you design, build, and automate workflows by connecting apps and services with a unique visual interface.

Unlike traditional automation tools, Make provides a visual canvas where you can see your entire workflow, making it easy to understand data flow and build complex scenarios without writing code.

  • 1500+ pre-built app integrations
  • Visual scenario builder with real-time execution
  • Advanced data transformation tools
  • Error handling and scenario monitoring

How Make.com Works

Build automations visually with Make's unique approach

1

Create a Scenario

Start with a blank canvas where you'll build your automation workflow. Each scenario represents a complete automation process that runs independently.

2

Add Modules

Drag and drop modules onto your canvas. Each module represents an app, service, or function. Connect triggers (events that start your scenario) with actions (tasks to perform).

3

Configure Connections

Set up secure connections to your apps and services. Make handles authentication and maintains these connections, so you only need to authorize once.

4

Map Your Data

Use Make's powerful mapping features to transform and route data between modules. See exactly how information flows through your scenario with visual data mapping.

5

Test and Deploy

Run your scenario in test mode to verify everything works correctly. Once satisfied, activate it to run automatically based on your triggers or schedule.

Key Make.com Concepts

Understanding these fundamentals will accelerate your automation journey

Modules

Modules are the building blocks of your scenarios. Each module performs a specific action like retrieving data, processing information, or triggering events. They connect visually to show data flow.

Example: Gmail module → Filter module → Slack module

Bundles

Bundles are packets of data that flow between modules. Each bundle contains the information processed by your scenario. Understanding bundles helps you optimize operations and costs.

Example: 1 email = 1 bundle, 100 emails = 100 bundles

Data Mapping

Data mapping allows you to connect outputs from one module to inputs of another. Make's visual mapping interface shows available data fields and lets you transform data between modules.

Example: Map {{email.subject}} to Slack message title

Routers & Filters

Routers split your workflow into multiple paths, while filters control which data passes through. Together, they create conditional logic that makes your automations intelligent and efficient.

Example: If priority = "high" → Send SMS, else → Send Email
